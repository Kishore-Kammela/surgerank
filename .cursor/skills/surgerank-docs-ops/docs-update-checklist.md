# Docs Update Checklist

## Before Editing
- [ ] Identify canonical target docs.
- [ ] Confirm audience and purpose of change.
- [ ] Check whether decision-log update is required.

## During Editing
- [ ] Preserve or add frontmatter metadata.
- [ ] Keep section order layered and scannable.
- [ ] Use tables for comparisons and thresholds.
- [ ] Add diagram captions/interpretation when using mermaid.

## After Editing
- [ ] Update cross-links and section README references.
- [ ] Validate consistency with scope and roadmap docs.
- [ ] Update `last_updated` where appropriate.
- [ ] Mark readiness status if document maturity changed.
