# Scope Documentation Package

## Purpose
Canonical scope package for SurgeRank V1, designed for markdown-first collaboration and PDF sharing.

## Audience
- Stakeholders and founders
- PM and operations
- Engineering and QA

## Read Order (for PDF export)
1. `01-executive-scope.md`
2. `02-product-scope.md`
3. `03-technical-scope.md`
4. `04-delivery-and-launch-plan.md`
5. `05-pricing-and-business-model.md`

## Document Set Summary

| Document | Primary Reader | Purpose |
| --- | --- | --- |
| `01-executive-scope.md` | Stakeholders | Business context, objective, and success definition |
| `02-product-scope.md` | PM + Engineering + QA | Scope boundaries, priorities, and acceptance criteria |
| `03-technical-scope.md` | Engineering + QA | Stack, architecture, data boundaries, NFRs |

| `06-risks-and-decisions.md` | PM + Stakeholders | Risk posture, mitigations, canonical decisions |
| `appendix-glossary.md` | All | Shared terms and abbreviations |

## Canonical Scope Position
- This package is the single source of truth for V1 scope and launch planning.
- Legacy `scope/*.docx` files are decommissioned after sign-off.
- Any scope change must be reflected here and in `docs/02-planning/decision-log.md`.
